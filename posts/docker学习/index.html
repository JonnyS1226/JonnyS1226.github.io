1. docker 相关命令

  • 虚拟化是一种资源管理技术，是将计算机的各种实体资源，如服务器，网络，内存以及存储等，予以抽象转换后呈现出来，打破实体结构间的不可切割的障碍，使用户可以用一定方式直接使用这些资源
  • 与传统虚拟机不同的是，容器是在操作系统层面实现虚拟化，直接复用主机的操作系统。而传统方式是在硬件层面实现。与传统方式相比，docker的优势在于启动速度快，占用体积小。
  • 以下命令是mac上的，windows或者linux上可能有略微不同。并且部分操作也可以通过客户端完成。

1.1. docker启动

launchctl start docker

1.2. 镜像相关命令

  • docker images 查看镜像情况
  • docker search xxx 搜索镜像情况
  • docker pull xxx 拉取某个镜像，默认使用latest版本
  • docker rmi id/name 根据id或者name删除镜像，这个镜像不能被容器正在使用

1.3. 容器相关命令

  • docker ps 查看正在运行的容器
  • docker ps -a 查看所有运行过的容器
  • docker run 镜像名:版本号 启动容器
    • -i：表示运行容器
    • -t： 表示容器启动后会直接进入其命令行
    • -name xxx / –name=xxx：为创建的容器命名
    • -v：表示目录映射关系（前者是宿主机目录，后者是映射到宿主机上的目录）
    • -d：创建守护式容器在后台运行
    • -p：端口映射，前者是宿主机端口，后者是容器内的映射端口
  • 交互式方式运行 -it：使用exit退出时就自动关闭了容器
  • 守护式方式运行 -id：使用docker exec -it id 进入容器，使用exit退出时不会自动关闭容器
  • docker stop id/name 停止容器
  • docker start id/name 启动已停止的容器
  • docker cp 需要拷贝的文件或目录 容器名称：容器目录： 拷贝文件到容器里
  • docker cp 容器名称：容器目录 需要拷贝到的文件或目录：从容器中拷贝文件出来
  • docker inspect 容器名称/id 可以查看容器相关信息，包括其ip
  • docker inspect --format='{{.NetworkSettings.IPAddress}}' name/id 可以通过format参数提取具体的信息，如这条命令就可以直接输出ip地址
  • docker rm id/name 删除容器（容器不能正在运行）

1.4. 目录挂载

docker run -di -v /usr/local/myhtml:/usr/local/myhtml --name=mycentos centos:7
  • 这样可以在创建容器的时候，将宿主机的目录与容器内的目录进行映射，这样就可以通过修改宿主机的某个目录的文件从而去影响容器
  • 若共享多级目录，可能会出现权限不足的提示，需要根据不同的os去添加参数，如centos需要添加--privileged=true 参数

1.5 迁移和备份

  • docker commit 容器名 保存为的镜像名： 将容器保存为镜像
  • docker save -o xxx.tar 镜像名： 备份，将镜像保存为tar文件
  • docker load -i xxx.tar ：从tar文件中导入镜像

2. Dockerfile

  • Dockerfile是由一系列命令和参数构成的脚本，这些命令应用于基础镜像（OS层面的， 如已经有了ubuntu或者centos）并最终创建一个新的镜像。

2.1 常用命令

命令 作用
FROM image_name:tag 定义了基于的基础镜像，启动构建流程
MAINTAINER user_name 声明镜像的创建者
ENV key value 设置环境变量
RUN command 是Dockerfile核心部分
ADD source_dir/file dest_dir/file 将宿主机的文件复制到容器内，如果是一个压缩文件，会自动解压
COPY source_dir/file dest_dir/file 和ADD相似，但是不自动解压
WORKDIR path_dir 设置工作目录

2.2 使用脚本构建镜像（以构建JDK为例）

FROM centos:7
MAINTAINER xxx
WORKDIR /usr
RUN mkdir /usr/local/java
ADD jdk-linux-x64.tar.gz /usr/local/java

ENV JAVA_HOME=/usr/local/java/jdk1.8.0.171
ENV JRE_HOME $JAVA_HOME/jre
ENV CLASSPATH $JAVA_HOME/bin/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib:$CLASSPATH
ENV PATH $JAVA_HOME/bin:$PATH
  • 写好Dockerfile后，执行docker build -t='jdk1.8' .，开始构建

3. Docker私有仓库

3.1 私有仓库搭建与配置

  • 作用于局域网下（内部）的镜像共享
  • docker pull registry 拉取私有仓库镜像
  • docker run -di --name=registry -p 5000:5000 registry 根据私有仓库镜像构建容器
  • 访问ip:5000/v2/_catalog， 可以看到一个json表示的内部镜像情况
  • 修改daemon.json，添加配置，insecure-registry:["ip:5000"]，这样配置使得信任这个ip的docker私有仓库，便于后续私有仓库镜像上传

3.2 上传和下载私有仓库镜像

  • docker tag name/id ip:5000/name 给镜像打一个标签
  • docker push ip:5000/name 上传私有仓库镜像，需要先把registry容器启动
  • 如果需要下载，也需要按照上述进行配置，然后直接docker pull ip:5000/name 即可
