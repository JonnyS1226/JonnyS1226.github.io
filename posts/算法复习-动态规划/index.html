算法复习-动态规划

January 18, 2020 2-minute read
算法设计与分析
算法设计与分析

算法简介

  • 理论角度：

    • 动态规划是重要算法，基本设计思想是将待求解问题分解成若干个子问题，分阶段求解子问题，前一阶段的子问题的解成为后续阶段子问题的解的计算信息，最后用这些子问题的最优解构造出原问题的最优解
    • 特征：子问题具有重叠性，有最优子结构
    • 基本步骤：找出最优解的特征，按最优解的性质来划分子问题和演算阶段，求出递推式，自底向上或自顶向下计算求值，根据局部最优解构造出整体最优解
      • 简单来说就是：设计状态 –> 求状态转移方程 –> 求出解

  • 个人理解：

    • 从图论角度，本质上是形成一个有向无环图，然后使用循环、记忆化搜索、DFS或BFS求解都可以
    • 从函数角度，动态规划其实这就像是高中数学的数列题的求解，求出递推式就成功了一大半

  • dp确实是相对比较困难的算法，为了加深学习，这里放上一个让我受益匪浅的讲述动态规划的博客：

  • 博文链接：Dynamic Programming Patterns
  • 上述博文作者将dp分为了5大类，基本涵盖大部分dp问题。主要是
    • 达成某个目标的最大/最小值
    • 达到目标的方法数
    • 区间合并dp
    • 字符串dp
    • 做决定问题，即当前元素是否要选择的问题
  • 而下面将根据博文中的分类，每一类选择1-2题进行分析其递推式或者状态转移方程。值得注意的是，每一类之间并非完全分离，比如背包问题，既可以作为第一类，也可以作为第五类来解决，但想法是相通的。

典型例题

1. 达成某个目标的最大最小值

1）题目描述

  • 题目描述常是：给出一个目标任务，找到实现这个任务的最大或最小开销/价值等

2）常见解决模式

  • 先选择出当前状态之前的某个状态（有些情况是选择出遍历之前的所有状态），查看其实现这个目标的最大/最小开销，然后加上从那个状态到当前状态的开销
  • 常见解决模式：$f(i)=min(f(i-1),f(i-2),…,f(i-k))+cost[i]$

3）例题–剪绳子问题

  • 题目描述：给定一段长的绳子，切成若干长度，求这切出的各段长的乘积最大为多少？
  • 分析：不妨设要求解的绳子长度为$n$，记$f(n)$表示长度为n的绳子题设条件下的最大乘积。
    • 这个问题则需要先查看绳子长度$L<n，($其中记 $L=n-i，0<i<n )$的绳子在题设条件下的最大乘积$f(n-i)$，然后从中选择出最大的。
    • 再乘以从那个状态到当前状态的“开销” $f ( i )$，就可以得到$f(n)$.
  • 根据上述分析，状态转移方程为：$f(n)=max(f(n-i)\times f(i))$

4）例题–01背包问题

  • 题目描述：给定背包承重量$W$，给出物品数$n$，各物品价值$V_i$，各物品重量$w_i$，求背包装物品得到的最大价值？
  • 分析：记承重为W，物品数为n的背包装物品最大价值为$f_n(W)$
    • 先查看在当前状态之前的状态的最大价值，记为$f_{n-1}(W-w_i)$和$f_{n-1}(W)$，
    • 再分别加上“开销”：$V_i$和$0$，取更大的一个。
  • 根据上述分析，状态转移方程为：$f_i(W)=max(f_{i-1}(W), f_{i-1}(W-w_i)+V_i)$

2. 达成目标的方法数

1）题目描述

  • 题目描述常是：给定要求，计算达成一个目标的方法数有多少

2）常见解决模式

  • 对所有可能的方法作和，即有向无环图的到某个点的路径数求和
  • 常见解决模式：$routes[i] = routes[i-1] + routes[i-2], … , + routes[i-k]$

3）例题–跳台阶问题

  • 题目描述：一次跳上1级台阶，也可以跳上2级台阶，求跳上一个n级台阶共有几种跳法
  • 分析：记$f(n)$表示跳n阶台阶的跳法数。
    • 跳1级台阶只有1种方法，跳2级台阶有2种方法。
    • 跳>2级台阶时，第一次可以跳一级，则一共跳法数是$f(n-1)$。第一次若跳两级，则是$f(n-2)$。
    • 故一共有$f(n-1)+f(n-2)$种。
  • 根据上述分析，递推式是：$f(n)=f(n-1)+f(n-2)$

4）例题–Unique Paths

  • 题目描述：摘自leetcode
  • 分析：设 $dp[i][j]$ 表示 $i \times j$ 的矩形网格下，从左上走到右下的总方法数。
    • 需要考虑全部的可行方法。考虑$ (i,j)$ 格，由于机器人只能向右或者向下走，若机器人走到该格上面一格，很显然就能走到这格。同理，若机器人走到该格左边一格，很显然也能走到这一格。
    • 所以到达 $ (i,j)$ 格的方法数是到达其上方和左方相邻方块的方法数之和
    • 其中对于 $dp[i][0]、dp[0][j]$ 需要初始化为1。因为这表示是一行或者一列，很显然只有一种走法。
  • 根据上述分析，递推式是： $dp[i][j]=dp[i-1][j]+dp[i][j-1]$

3. 区间合并DP

1）题目描述

  • 题目描述常是：在一段区间（或看成数轴）中，考虑当前和左右位置，合并小区间，来找到问题最优解

2）常见解决模式

  • 对每个二维层面（即看成数轴或区间）上的每一段满足条件的间隔进行分析，得到合并小区间的最优解，进而得到整个区间的最优解
  • 常见解决模式：$dp[i][j] = dp[i][k] + result[k] + dp[k+1][j]$，$k$ 从 $i$ 到 $j$

3）例题–矩阵链乘法

  • 题目描述：给定n个矩阵（ $A_1,A_2,…,A_n$ ），矩阵 $A_i$ 的规模为 $p_{i-1} \times p_i$，求以何种次序进行运算（即通过不同的添括号方法），会使得矩阵乘法 $A_1A_2…A_n $所需要的标量乘法次数最少？

  • 分析：记 $dp[i][j]$ 表示矩阵乘法 $A_iA_{i+1}…A_j$所需要的标量乘法次数的最小值。

    • 可以将这个问题看成区间dp，对于$dp[i][j]$对应的 $A_iA_{i+1}…A_j$，假设一个最优分割点在 $A_k$和$A_{k+1}$之间，则有 $dp[i][j]=dp[i][k]+p_{i-1}p_kp_j+dp[k+1][j]$
    • 而需要对整个区间所有满足条件间隔进行分析，然后取最优值

  • 根据上述分析，递推式如下

    • $$ dp[i][j]= \begin{cases} 0, &\text{if i=j}\
      \mathop{min}\limits_{i \leq k<j}({dp[i][k]+dp[k+1][j]+p_{i-1}p_kp_j}), &\text{if i<j} \end{cases} $$

4. 字符串DP

1）题目描述

  • 常常是关于两个字符串（有限长度）之间的比较，修改，或者子串判定相关的dp问题

2）常见解决模式

  • 主要是通过遍历两个字符串，若相同字符则转向一个情况，不同转向另一个情况

3）例题–最大公共子序列长度

  • 题目描述：给定两个字符串，求出它们的最长公共子序列的长度。

  • 分析：设 $dp[i][j]$ 表示对于字符串 $s_1[1…i]$ 和 $s_2[1…j]$，它们的最大公共子序列长度。设 $s_1[i]$表示 $s_1$中的第$i$个字符，$s_2$同理

    • 先初始化，显然 $dp[0][j]$ 和 $dp[i][0]$ 都初始化为0
    • 很显然，若$s_1[i] = s_2[j]$，则最大公共子序列长度很显然是之前的最大公共子序列长度＋1
    • 若 $s_1[i] \neq s_2[j]$，则最大公共子序列长度是 $s_1[i-1]$和$s_2[j]$ 的最大公共子序列长度以及 $s_1[i]$ 和 $s_2[j-1]$的最大公共子序列长度中的更大值

  • 根据上述分析，递推式如下：

    • $$ dp[i][j]= \begin{cases} dp[i-1][j-1]+1, \ \text{if $s_1[i-1]=s_2[j-1]$}\
      max(dp[i-1][j],dp[i][j-1]),\ \text{if $s_1[i-1] \neq s_2[j-1]$ } \end{cases} $$

5. 做决定问题

1）题目描述

  • 关于当前元素/状态取舍，基于一定目标，做出决定的问题

2）常见解决模式

  • 如果取当前元素，则需要更新临时dp数组；如果不取当前元素，则dp数组还是放置前一个元素时的值。在这些情况中，选择最优解

  • 常见解决模式如下：

    // i - indexing a set of values
// j - options to ignore j values
for (int i = 1; i < n; i++) {
   for (int j = 1; j <= k; j++) {
       dp[i][j] = max({dp[i][j], dp[i-1][j] + arr[i], dp[i-1][j-1]});
       dp[i][j-1] = max({dp[i][j-1], dp[i-1][j-1] + arr[i], arr[i]});
   }
}

3）例题–01背包问题

  • 背包问题是这一类很典型的问题，同时也符合第一类，在第一类中已有[阐述](####1. 达成某个目标的最大最小值)

4）例题–最大子段和问题

  • 题目描述：求一个数组中，某连续一段子数组求和，求这个和的最大值？（也即股票买卖最佳时机问题，事实上看成股票问题更容易理解对应的算法）

  • 分析：设 $dp[i][j]$ 表示在索引为 $i$ 的这一天，持股状态为 $j$ 的最大利润。$j=0$ 表示不持股，且强调是卖出股票后的不持股状态；$j=1$ 表示持股.

    • 根据上述假设，显然$j$ 只能从1变为0，不能从0变为1
    • 对于不“取“这个元素，即 $j=0$，则考虑前一天也不持股，或者前一天持股而今天卖出，这两种情况中取最优
    • 对于”取“这个元素，即 $j=1$，则要通过”收益”看，是维持这个状态不持股，还是选择这个元素
    • 由于最晚到最后一天一定要卖出，所以最后返回 $dp[length-1][0]$ 即结果

  • 通过上述分析，递推式为

    • $dp[i][0] = max(dp[i - 1][0], dp[i - 1][1] + prices[i])$
    • $dp[i][1] = max(dp[i - 1][1], -prices[i])$
