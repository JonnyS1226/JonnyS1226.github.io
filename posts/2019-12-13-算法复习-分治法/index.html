算法简介

分治策略是把一个较大的问题分解成几个与原问题相似的子问题，找到求出这几个子问题的解法后，再以适当的方法组织合并成完整问题的解法。

分治策略往往使用递归解决问题，主要步骤分为：分解，解决，合并。故要使用分治策略，就需要适当地分解，同时设置好递归出口。

典型例题

1. 找出数组中重复数字

1) 题目描述

在一个长度为n+1的数组里的所有数字都在1-n的范围内，所以数组中至少有一个数字是重复的，找出一个重复的数字。

2) 分析

  • 可以使用排序寻找(即蛮力法)，但时间复杂度较高。
  • 使用Hash函数保存遍历的数字降低了时间复杂度，但空间复杂度较高。
  • 使用分治策略，降低了时间和空间复杂度：
    • 先确定一个中间值t，对于比t小的和比t大的进行分治考虑，重复的值一定在这两块范围中的某一块。
    • 判断是否有重复：统计某范围内数字出现个数count，若count>该范围长度，则该范围一定有重复值。于是对该范围进行(递归)分治。
    • 递归出口是当某一范围局限到一个数字时，则说明该数字就是重复值；当然也可以不使用递归。
    • 需要指出的是，这种策略不能找出所有的重复数字。

3) 代码

    // 递归
    public static int recursiveToFind(int[] array, int start, int end, int length)
    {
        if(end == start)
        {
            return start;
        }
        int middle = (end + start) / 2;
        int count = rangeCount(start, middle, array, length);
        if (count > (middle - start + 1))
        {
            return recursiveToFind(array, start, middle, length);
        }
        else {
            return recursiveToFind(array, middle + 1, end, length);
        }
    }

    // 非递归
    public static int noRecursiveToFind(int[] array, int length)
    {
        int start = 1;
        int end = length - 1;
        while (end >= start)
        {
            int middle = (end + start) / 2;
            int count = rangeCount(start, middle, array, length);
            if (end == start)
            {
                return start;
            }
            if (count > (middle - start + 1))
            {
                end = middle;
            }
            else {
                start = middle + 1;
            }
        }
        throw new IllegalArgumentException("没有重复的");
    }

    //范围统计数字个数
    public static int rangeCount(int start, int end, int[] array, int length)
    {
        if(length == 0)
        {
            return 0;
        }
        int cnt = 0;
        for (int i = 0; i < length; i++)
        {
            if(array[i] <= end && array[i] >= start)
            {
                cnt++;
            }
        }
        return cnt;
    }

4) 复杂度分析

  • 主要分析时间复杂度：
    • 非递归方式：每次都对半分，按照二叉树顶点数n与树高h之间的关系: $n=2^h-1$，有$T(n)=log_2(n+1)$，所有可以认为要调用rangeCount$O(logn)$次，而rangeCount的时间复杂度为$O(n)$，故总时间复杂度为$O(nlogn)$， 空间复杂度$O(1)$
    • 递归方程：$T(n)=T(\frac{n}{2})+O(n)$， 空间复杂度$O(1)$

2. 最大子数组

1) 题目描述

给定n个元素的整数数组，有正有负，求：和最大的连续子数组（子段和）。

2) 分析

  • 暴力法：对于n长度的数组，显然最多有$C^2_n$种情况，即时间复杂度可认为为$O(n^2)$
  • 动态规划：在后续章节展示，效率高。
  • 分治策略：将数组折半分为左右两部分A, B， 对于最大子段和，一定是：要么在A内，要么在B内，要么跨越A,B。由此将整个问题分为三部分进行分解处理。
    • 对于跨越情况的最大值，如Array[low, mid]和Array[mid+1, high]，由于最大和子段的左右边界$i, j$满足$low\leq i\leq mid \leq j \leq high$， 则先计算[low, mid]的连续和最大值和左边界限$i$，再计算[mid+1, high]的连续和最大值和右边界限$j$，再作和即得到”跨越“最大值和$i,j$。
    • 递归出口是low==high，此时返回array[low]。

3) 代码

 	public static int findMaxSubArrayCrossing(int[] array, int low, int high, int mid)
    {
        int leftSum = 0;
        int tmpLeftSum = 0;
        int maxLeftLocation = 0;
        for (int i = mid; i >= low ; i--) {
            tmpLeftSum += array[i];
            if (tmpLeftSum > leftSum)
            {
                leftSum = tmpLeftSum;
                maxLeftLocation = i;
            }
        }
        int rightSum = 0;
        int tmpRightSum = 0;
        int maxRightLocation = 0;
        for (int i = mid + 1; i <= high ; i++) {
            tmpRightSum += array[i];
            if (tmpRightSum > rightSum)
            {
                rightSum = tmpRightSum;
                maxRightLocation = i;
            }
        }
        return rightSum + leftSum;
    }

    public static int findMaxSubArray(int[] array, int low, int high){
        if (high == low)
        {
            return array[low];
        }
        int mid = (low + high) / 2;
        int leftSum = findMaxSubArray(array, low, mid);
        int rightSum = findMaxSubArray(array, mid + 1, high);
        int crossingSum = findMaxSubArrayCrossing(array, low, high, mid);
        if (leftSum >= rightSum && leftSum >= crossingSum)
        {
            return leftSum;
        }
        else if (rightSum > leftSum && rightSum > crossingSum) {
            return rightSum;
        }
        else {
            return crossingSum;
        }
    }

4) 复杂度分析

  • 主要分析时间复杂度：

$$ T(n) = \begin{cases} O(1), &\text{if $n$=1}\
2T(\frac{n}{2})+O(n)), &\text{if $n$>1} \end{cases} $$

  • 根据主定理可以得到，$T(n)=O(nlogn)$。
